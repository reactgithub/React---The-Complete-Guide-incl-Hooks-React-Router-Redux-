create-react-app

A tool for creating react applications with no build configuration.

Installing create-react-app

    npm install -g create-react-app

Creating a new React application

    create-react-app my-app
    cd my-app
    npm start

Open http://localhost:3000/

Creating a minified bundle.

npm run build

Setting up the react-complete-guide project

    create-react-app react-complete-guide --scripts-version 1.1.5

Navigate to the new folder

    cd react-complete-guide/

Start the Project

    npm start

Be sure to leave this running while working on the application so updates are performed automatically.

React.createElement()

A method for rendering an element that takes three or more arguments, the first being the html element, the second being the configuration element (often null), and the third is the children elements.

Typically elements are rendered using JSX, not the React.createElement() method.

className

In JSX, CSS styling is added using className, not class. This is because class is a reserved word in JavaScript.

JSX Best practices

  • JSX must have a single root element.
  • Folders and files containing JSX components begin with capital letters.
  • All elements starting with lower case are reserved for native HTML elements.
  • Only allows embedding of JavaScript expressions.

props

The properties that are added onto a component.

Children

Anything that is between the open and clising tags of the element.

state

  • Only availble inside class based components (extends Component).
  • If state changes, React will update the DOM and re-render all components.

React Event Handlers

  • In React, event handlers are camel case, such as onClick instead of onclick.
  • It's a convention to name event handler functions ending with 'Handler' such as switchNameHandler().
  • parenthesis are left off of event handler function calls because they would cause the function to be called immediately instead of waiting for the event.

useState

  • Allows the use of state in a functional component.

useState

  • Allows state to be managed in a functional component.
  • Returns an array with two elements.
  • The first element returned is the current state.
  • The second element is a function that allows us to update the state such that React is aware of it and will re-render the component.
  • Can't use 'this' when using useState because it only exists in class based components.
  • useState (React Hooks) will not merge existing state with the new state. It will just overwrite the old state with the new state in entirety.
  • useState does not need to pass an object, it could be a string, array, number, boolean, etc.
  • There can be however many useState() calls in a page, referring to different state objects.

Passing Method References Between Components

This is a pattern that is used to call a method in one component from another component.

Pass a reference to a handler as a property to a component.

Two ways to pass data

  • The first is to call bind(this) on the name of the function that is passed as a reference (this is more performant and should be used when possible)
  • The second uses an arrow function like so: onClick={() => this.myFunctionHandler('myNewValue')}
